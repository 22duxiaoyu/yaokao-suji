# 免费部署指南

目标：使用 Vercel Hobby、Supabase Free 和 GitHub Releases 发布一个低成本面试 Demo。

## 1. Supabase

1. 创建一个 Free Project。
2. 在 Storage 创建名为 `materials` 的私有 Bucket。
3. 在 Project Settings 获取：
   - Project URL
   - Service Role Key
   - PostgreSQL Pooler Connection String
4. 不要把 Service Role Key 写进 `NEXT_PUBLIC_*` 变量。

本地临时设置 Supabase 数据库连接后执行：

```bash
DATABASE_URL='你的 Supabase Pooler 连接串' npm run db:migrate
DATABASE_URL='你的 Supabase Pooler 连接串' npm run db:seed
```

## 2. Vercel

从 GitHub 导入仓库，Framework 选择 Next.js，配置环境变量：

```env
DATABASE_URL=Supabase Pooler连接串
DATABASE_POOL_MAX=1

SUPABASE_URL=https://xxxx.supabase.co
SUPABASE_SERVICE_ROLE_KEY=服务端Service Role Key
SUPABASE_STORAGE_BUCKET=materials

AI_PROVIDER=deepseek
AI_API_KEY=DeepSeek生产演示Key
AI_MODEL=deepseek-v4-flash
AI_BASE_URL=https://api.deepseek.com

PUBLIC_DEMO_MODE=true
NEXT_PUBLIC_DEMO_MODE=true
DEMO_AI_DAILY_LIMIT=20
DEMO_MAX_GENERATED_CARDS=6
DEMO_MAX_UPLOAD_MB=4
NEXT_PUBLIC_DEMO_MAX_UPLOAD_MB=4
```

部署完成后验证：

```text
https://你的项目.vercel.app/api/demo/status
```

预期返回：

```json
{
  "ok": true,
  "mode": "portfolio-demo",
  "services": {
    "database": "database",
    "fileStorage": "supabase",
    "ai": "deepseek"
  }
}
```

## 3. Demo 使用约束

- 推荐上传 4MB 以下的 PDF、docx、txt。
- 面试前提前打开一次 Supabase 项目，避免免费项目休眠后的首次唤醒等待。
- DeepSeek 账户只保留小额余额。
- 不使用真实患者信息、个人隐私资料或受限版权文件测试。
- 准备一份粘贴文本，文件上传受网络影响时可以立即切换演示。

## 4. Android APK

部署后运行：

```bash
cd android-webview
APP_URL=https://你的项目.vercel.app/ ./build-apk.sh
```

把 `dist/yaokao-suji-demo.apk` 上传到 GitHub Releases。官网或 README 的下载按钮链接到该 Release Asset。

## 5. GitHub 发布前检查

```bash
npm run check
rg -n "API_KEY|SERVICE_ROLE|postgres://" . --glob '!node_modules/**' --glob '!.next/**' --glob '!.env.local'
```

确认以下文件不会提交：

- `.env.local`
- Android keystore
- `uploads/` 用户文件
- `.next/`
- 本地数据库信息

## 6. 中国大陆访问说明

Vercel 和 Supabase 的免费服务不保证中国大陆网络质量。作品集应同时准备：

- 两分钟本地演示视频
- README 截图
- 本地运行命令
- Android APK

在线服务不可用时仍然可以完整讲清产品和技术链路。
